Introduction

A CLI for working with OpenCLI Specs

ocli is a command line interface designed to make working with OpenCLI Spec documents easier. It provides a number of capabilities, including:

  • validating OpenCLI Spec documents
  • generating Documentation from OpenCLI Spec documents

The commands are documented below. You can also find out more about each command using the contextual --help flag. e.g.:

ocli gen --help

$ ocli check <path-to-spec>

Check an OpenCLI Spec document for errors

The check command will validate an OpenCLI Spec document and return any errors that are found.

Arguments
<path-to-spec>
stringrequired

The path to the OpenCLI Spec document to check

this command will check the validity of the given OpenCLI document by parsing it and checking it against the OpenCLI Specification.

[!TIP] The format of the input file will be determined by the file extension.

$ ocli gen {command} <arguments> [flags]
group non-executable

Commands used to generate code/docs from an OpenCLI Spec document

$ ocli gen docs <spec-file> [flags]

Generate documentation from an OpenCLI Spec document

The generate docs command will generate documentation from an OpenCLI Spec document. You can specify the format of the documentation to be generated using the --format flag.

Arguments
<spec-file>
stringrequired

The path to the OpenCLI Spec that documentation will be generated from

Flags
--out-o
stringrequireddefault: ./docs

The path to the directory where the generated docs will be output

--format-f
stringenumrequired

The format of the documentation to generate

Accepted values
  • markdownPlain text using standard [Markdown](https://commonmark.org).
Alternative sources
  • env: $OCLI_DOCS_FORMAT
  • file: $.docs.format
--footer
booleandefault: true

Include the footer in the docs

Alternative sources
  • env: $OCLI_DOCS_FOOTER
  • file: $.docs.footer
--dryrun
booleandefault: true

When true the docs contents will be output to stdout instead of the file
