# Security Policy

## Reporting Vulnerabilities

If you discover a security vulnerability in this project, please create a GitHub Issue in this repository and include "Security" in the issue title.

To help maintainers triage quickly, include affected files or components, clear reproduction steps, expected vs. actual behavior, impact, environment details, and (if possible) a minimal proof of concept.

Please do not include secrets, private keys, tokens, or sensitive personal data in issue content or attachments.
